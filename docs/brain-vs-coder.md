# 大脑 vs 小脑：七步流水线分工图

## 总纲

| | 大脑（免费 Gemini/DeepSeek） | 小脑（Codex 桌面版） |
|---|---|---|
| 角色 | 项目经理 + 架构师 | 写代码的执行者 |
| 成本 | 免费 | 付费（精准花） |
| 做的事 | 分析、规划、出方案 | 写代码、跑测试、改 bug |

## 七步分工

```
┌─────────────────────────────────────────────────────────┐
│              大脑（免费网页模型）                          │
│                                                         │
│  第 ① 步：需求磨透          ← 提示词①                    │
│  第 ② 步：定死技术架构      ← 提示词② + 输出 claude.md  │
│  第 ③ 步：生成开发计划      ← 提示词⑦                    │
│  第 ⑥ 步（分析）：分析报错  ← 提示词④（只做分析不出代码）│
│                                                         │
│  产出：PRD + 架构设计 + 原子化开发计划 + 修复指令         │
└──────────────────────┬──────────────────────────────────┘
                       ↓
                ┌──────┴──────┐
                │  第 ④ 步    │
                │  人工审计    │  ← 可有可无，大脑小脑均不参与
                └──────┬──────┘
                       ↓
┌─────────────────────────────────────────────────────────┐
│              小脑（Codex 桌面版）                         │
│                                                         │
│  第 ⑤ 步：分阶段写代码     ← 按开发计划，分对话分 Task   │
│  第 ⑥ 步（执行）：外科手术  ← 按大脑给的修复指令改代码    │
│  第 ⑦ 步：测试自愈循环     ← 提示词⑥，写测试→跑→修复    │
│                                                         │
│  随时调用：                                            │
│    提示词③ — 代码审计与重构（清剿硬编码/大文件/静默失败）│
│    提示词⑤ — 前端指令翻译官（升维视觉到 Apple/Linear 级）│
└─────────────────────────────────────────────────────────┘
```

## 第 ⑤⑥⑦ 步的闭环细节

### 第 ⑤ 步：写代码

```
大脑出开发计划 ──→ 开新对话框 ──→ 丢给 Codex 按 Task 写
                      ↓
               每写完一个 Task 就 git commit
                      ↓
               拉起服务，看能不能跑
                      ↓
              能跑？→ 进第 ⑦ 步
              报错？→ 进第 ⑥ 步
```

### 第 ⑥ 步：调 BUG

```
报错日志
    ↓
┌─ 大脑（Gemini + 提示词④）──┐
│ PROBE → SURGERY → VERIFY    │  ← 分析根因，不出代码
│ 输出：@文件名 + 改哪行 + 怎么改 │
└──────────┬─────────────────┘
           ↓ 复制修复指令
┌─ 小脑（Codex）──────────────┐
│ 按指令改代码，不改其他       │  ← 外科手术式修复
│ 跑测试验证                   │
└─────────────────────────────┘
```

### 第 ⑦ 步：测试自愈循环

```
┌─ 小脑（Codex + 提示词⑥）───┐
│  写 Playwright 测试脚本      │
│  ↓                          │
│  跑测试                      │
│  ↓                          │
│  ┌─ 失败 ─→ 自愈分析 ──┐   │
│  │  是测试脚本问题？      │   │
│  │  → 自己改脚本重跑      │   │
│  │  是代码 bug？          │   │
│  │  → 回到第 ⑥ 步走大脑   │   │
│  └───────────────────────┘   │
│  ↓                          │
│  全绿 ✅                     │
└─────────────────────────────┘
```

## 提示词速查表

| 编号 | 用途 | 在谁那跑 | 调用时机 |
|---|---|---|---|
| ① | 需求完善 → PRD | 大脑 | 第 ① 步 |
| ② | 系统架构 → claude.md | 大脑 | 第 ② 步 |
| ③ | 代码审计与重构 | 大脑分析，小脑执行 | 随时 |
| ④ | BUG 调试调度引擎 | **大脑分析**，小脑执行 | 第 ⑥ 步 |
| ⑤ | 前端指令翻译官 | 大脑出指令，小脑执行 | 写前端时 |
| ⑥ | 浏览器测试自愈 | **小脑闭环** | 第 ⑦ 步 |
| ⑦ | 开发计划生成 | 大脑 | 第 ③ 步 |
