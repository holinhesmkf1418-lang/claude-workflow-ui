# 四条保命红线

> AI 写代码最喜欢「自由发挥」，不立规矩，项目必崩。这四条用血泪换来，直接抄进项目的 `claude.md`。

---

## 红线一：零硬编码

**坚决禁止业务逻辑硬编码。** AI 最爱把规则写死成一堆 if-else。

**要求：**
- 所有业务规则、状态字典、正则表达式、阈值必须做成配置化或数据库驱动
- AI 的活是写一个健壮的引擎去读配置，不是把规则焊死在代码里
- 前端颜色、间距、圆角全部提取为 CSS Variables（Design Tokens）

---

## 红线二：强制结构化输出

**前后端交互别让 AI 自由发挥。**

**要求：**
- 用 JSON Schema / Pydantic / Zod 定死数据结构
- 少个字段、乱改层级，立马打断重构
- 系统提示词必须工程化、结构化

---

## 红线三：死盯范围遗漏

**处理多层级数据时，AI 图省事经常丢层级关联。**

**要求：**
- 合并链路，死也不能丢标识（如单位工程与分部工程的物理标识融合）
- 强制保留并校验所有数据层级标识
- 执行合并而非覆盖，确保底层数据绝对完整

---

## 红线四：严禁静默失败

**AI 特爱用 `try-catch` 把错误悄悄吞掉。**

**要求：**
- 所有异常显式抛出、记录
- 数据入口用强校验拦住脏数据（Pydantic v2 / Zod）
- 移除所有裸 `try-catch` / `except: pass`
- 建立显式、精确的错误冒泡与捕获机制
