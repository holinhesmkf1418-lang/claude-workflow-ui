# AI 编程 SOP：从想法到交付的完整流水线

> 核心心法：AI 把「做出来」的成本打到了地板，稀缺的不再是「会写代码」，而是「知道该做什么、怎么拆、怎么控、怎么验收」。

---

## 一、双脑工作流

| 角色 | 作用 | 工具 | 成本 |
|---|---|---|---|
| **大脑**（做规划） | 需求推演、架构设计、生成计划 | 免费网页版强推理模型（Gemini 3.1 Pro / DeepSeek） | 免费 |
| **小脑**（写代码） | 按计划执行、写代码、调 bug | Codex 桌面版 / Claude Code CLI | 付费（精准花） |

**铁律**：前 4 步（需求→架构→计划→审计）在免费网页模型上跑，绝对不进编程终端烧 token。

---

## 二、七步流水线

### 第 1 步 · 把需求磨透
**提示词①：需求完善**
代入「资深产品经理 + 业务架构师」，跟 AI 反复磨需求。好的需求提示词会做"需求推演"——你只说了要个前端页面，它自动补全后台配置、权限管理、异常处理、数据看板。

### 第 2 步 · 定死技术架构
**提示词②：系统架构设计**
根据需求出系统架构，把前后端框架、数据库、编码规范全部定死。核心产物——项目专属 `claude.md`（300 字以内"家规"）。

### 第 3 步 · 生成开发计划
**提示词⑦：首席架构师 + 敏捷交付**
把 PRD + 架构产出物喂给提示词，生成可勾选 `[ ]` 的原子化开发计划。每个任务 2-4 小时干完，分四个阶段：基建数据模型 → 后端 API → 前端交互 → 测试联调。

### 第 4 步 · 人工审计计划
计划文档必须自己审。结尾让 AI 生成启动命令.md，也可以把整份计划丢回 Gemini 评估优化。

### 第 5 步 · 分阶段写代码
进编程终端，按计划一个阶段开一个对话框写代码。写完后按启动命令.md 拉起所有服务。

### 第 6 步 · 调 BUG
**提示词④：BUG 调试调度引擎**
报错后收集日志，去免费的 Gemini 用 BUG 调试提示词拿到精准修复指令，再回终端执行。走「探测→精准定位→外科手术式局部修复→回归验证」。

### 第 7 步 · 自动化测试自愈
**提示词⑥：浏览器自动化**
让 AI 自己陷入「测试-修复-验证」的死循环，直到跑出正确结果。

## 三、随时调用的弹药

| 提示词 | 用途 | 说明 |
|---|---|---|
| ③ 代码审计与重构 | 初版跑通后必须审计 | 清剿大文件、硬编码、静默失败 |
| ⑤ 前端指令翻译官 | 前端 UI 治理 | 把自然语言翻译专业指令，升维到 Linear/Apple 质感 |

## 五、成本控制

- 善用上下文缓存（Context Caching）
- 指令要轻、要准——只给相关代码切片
- 调试必须指向确定的文件名、函数名，禁止全局乱搜
- 每次对话提交都会重读历史，所以指令尽量一次说全

## 六、Git 纪律

- 原子化提交：每次让 AI 跑测试前先 git commit
- 严禁 AI 擅自大重构；小 bug 指定修改边界

## 七、变现方向

1. 给传统企业做内部降本工具
2. 垂直行业小工具卖给同行
3. 给有想法没技术的人做 MVP
4. SOP 和提示词做知识付费/陪跑

## 八、工具选型

- **优先**：Codex 桌面版 + 会员
- **国内可用**：Claude Code CLI
- **不推荐**：Cursor、Trae（复杂项目/MCP 把控不足）
